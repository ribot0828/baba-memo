# 🗒️ 馬場メモ

競馬の**項目別メモ**（開催場・馬名・自由項目）を、開催日に関係なく残せる単一HTMLアプリ。
サーバ不要・データは端末内のみ・外部送信なし。

[馬場カレンダー](https://github.com/ribot0828/baba-calendar)というカレンダー型の馬場メモアプリから、
「一般メモ」機能だけを取り出して独立させた軽量版です。開催日・開催スケジュールを気にせず、
馬名の系統整理や競馬場ごとの傾向メモだけをシンプルに使いたい人向け。

> **⚠️ 非公式のファンツールです。** JRA（日本中央競馬会）・関連団体とは一切関係なく、公認・提携もされていません。詳しくは[免責事項](#免責事項非公式について)をご覧ください。

## 使い方

`index.html` をブラウザで開くだけ（PC・スマホ両対応）。ダブルクリック、またはスマホのブラウザで開く。

### 項目の種類
- **開催場**（JRA10場から選択）
- **馬名**（日本語名。**大文字小文字を問わず同一馬として自動でまとまる**／英名優先で判別）。馬名では以下も記録可：
  - **系統**（任意。英名は「；」の後に併記可。例: `ロベルト；Roberto`）
  - **見なし系統**（任意。公式の血統情報が分からなくても、自分で仮に系統付けできる自由記入欄。例: ロベルト系）
- **自由**（海外・地方など自由入力。例: ロンシャン、大井、凱旋門賞）

### メモの追加
右下の「＋」→ ①種類（開催場／馬名／自由）を選び、項目名を入力・選択 → ②メモ本文を入力して保存。
1つの項目に複数のメモを追加できます。画面上部の検索窓で、項目名・英名・系統・見なし系統・メモ本文・馬場・馬場状態を横断検索（大小文字問わず）できます。

### 馬場・馬場状態・距離（開催場／自由のみ・すべて任意）
「開催場」「自由」の項目メモには、次の3つを任意で追加記録できます（馬名には追加されません）。
- **馬場**: 芝 / ダート / 障害（チップをもう一度押すと解除）
- **馬場状態**: 良 / 稍重 / 重 / 不良（同上）
- **距離**: 1000〜3600mの100m刻みで選択（空欄可）

値を設定したメモはカード上に馬場チップ・状態・距離が表示され、検索対象にもなります。

### クッション砂産地割合の記録（ダート限定・任意）
「開催場」「自由」のメモで馬場を**ダート**に設定すると、編集シートに「クッション砂産地割合（任意）」欄が表示されます。施工年月日と、産地（青森／新潟／宮城／福島／愛知／宮崎／豪アルバニー／その他（自由入力））＋割合%の行を「＋行を追加」で複数登録できます（合計が100%にならなくても保存可）。

登録した記録はメモとは別に保存され、項目カードに最新の産地割合バー（白砂%つき）と「産地割合の推移を見る」トグルで一覧表示されます。各記録は個別に削除可能です（確認ダイアログあり）。同じ項目・同じ施工年月日で再保存すると上書き更新されます。

## インストール（PWA・アプリ化）

GitHub Pages等のURLで開くと、**アプリとしてインストール**できます（審査不要・無料）。

- **iPhone(Safari)**: 共有 → 「ホーム画面に追加」。
- **Android(Chrome)**: メニュー → 「アプリをインストール」/「ホーム画面に追加」。
- **PC(Chrome/Edge)**: アドレスバー右のインストールアイコン。

インストール後はアイコン起動・全画面表示・**オフラインでも起動**（Service Workerがアプリ本体をキャッシュ）。ページ本体はオンライン時にネットワーク優先で取得するため、更新も自動で反映されます。
※ 構成ファイル: `manifest.webmanifest`, `sw.js`, `icon-192/512/180.png`。アプリ更新時は `sw.js` の `CACHE` 名（`baba-memo-v1`→`v2`…）を上げてください。

## データ入出力（可逆・馬場カレンダーと完全互換）

- **CSV**（推奨・Excel等で編集可）: 列は
  `id,kind,subject,subjectEn,lineage,presumedLineage,tags,memo,createdAt,updatedAt,surface,condition,distance`
  - `kind`: 種類。`venue`（開催場）/ `horse`（馬名）/ `other`（自由）のいずれか。
  - `lineage`（系統）・`presumedLineage`（見なし系統）は馬名(`horse`)のみ有効。
  - `surface`（馬場: 芝/ダート/障害）・`condition`（馬場状態: 良/稍重/重/不良）・`distance`（距離: 1000〜3600の100刻みの数値）は開催場(`venue`)・自由(`other`)のみ有効・すべて任意。
  - インポート時、`id` が既存と一致する行は **上書き更新**、無ければ新規追加（重複しない＝可逆）。
  - **新規追加なら `id` 列は不要**。必要な列は `subject` だけで、他は任意。
  - 旧バージョン（`surface`/`condition`/`distance` 列が無いCSV）もそのまま読み込め、新3列は空欄として扱われる。
  - 「CSVテンプレDL」で記入例つきの簡易テンプレを取得可能。
  - **「🤖 AIでCSV作成」**：手入力が大変なとき用。プロンプトをコピーしてChatGPT/Claude等に自分のメモと一緒に貼ると、取り込めるCSVを生成してくれる。
- **JSON**: 完全バックアップ用。`{notes:[...], sand:[...]}` 形式（`sand` はクッション砂産地割合の記録）。旧バージョンの `sand` キーが無いJSONもそのまま読み込める。
- **項目で絞り込んで保存**: 全件エクスポートに加え、**種類・項目**で絞り込んだ分だけCSV/JSONで書き出せる（該当件数を表示）。JSONには該当項目の砂記録も含まれる。

### クッション砂産地割合（`sand`）のCSV形式

「砂記録をCSVで保存」で書き出せる、メモCSVとは別形式のCSV（縦持ち・1産地=1行）。インポート欄はヘッダーで自動判別するため、メモCSVと同じファイル選択入力からそのまま読み込める。

- 列: `kind,subject,subjectEn,date,origin,percent`
  - `kind`: `venue`（開催場）/ `other`（自由）のいずれか。
  - `subject`/`subjectEn`: 項目名（メモと同じ）。
  - `date`: 施工年月日（`YYYY-MM-DD`）。
  - `origin`: 産地名（例: 青森、豪アルバニー）。
  - `percent`: 0〜100の数値。
  - 同一 `(kind, subject/subjectEn, date)` の複数行は、取り込み時に1つの砂記録（産地×割合の組み合わせ）としてまとめられる。`id` 列は不要。
  - インポートは既存の砂記録と `(項目キー, date)` が一致すれば上書き更新、無ければ新規追加。
  - 「砂CSVテンプレDL」で記入例（中山 / 2026-06-01 / 青森90% と 豪アルバニー10% の2行）を取得可能。
  - CSVインジェクション対策（`= + - @` で始まる産地名の `'` 付与・除去）はメモCSVと同じ仕組みを利用。
  - 「馬場カレンダー」アプリの砂記録CSVと**完全に同一形式**で、相互に読み込み可能。

### 「馬場カレンダー」とのデータ互換性

本アプリのメモCSV/JSON列定義・砂記録CSV形式は、[馬場カレンダー](https://github.com/ribot0828/baba-calendar)アプリの「一般メモ」機能と**完全に同一**です。

- 馬場カレンダーの「一般メモ」タブでエクスポートしたメモCSV/JSON・砂記録CSVは、そのまま馬場メモに読み込めます。
- 逆に、馬場メモでエクスポートしたメモCSV/JSON・砂記録CSVも、馬場カレンダーの「一般メモ」タブにそのまま読み込めます。
- クッション砂産地割合の記録（`sand`）はどちらのアプリでも一般メモに紐づくユーザー記録として同じ形式で扱われます（馬場カレンダーが内蔵する読み取り専用のダート砂産地割合データとは別物・連動しません）。
- ※ localStorageの保存キーはアプリごとに異なるため（本アプリは `babaMemo.v1`）、データの引き継ぎはCSV/JSONのエクスポート・インポートを介して行ってください（自動移行はしません）。

日付は `YYYY-MM-DD` 形式。

## セキュリティ / 配布

コミュニティ配布を想定した設計：

- **サーバなし**の単一HTML → バックエンド侵害リスクなし。GitHub Pages等に静的ファイルとして置くだけで配布可能。
- **外部通信を全面遮断**（CSP `connect-src 'none'`）→ メモが外部に送信されない。トラッキングなし。
- **外部CDN不使用** → サプライチェーン汚染の余地なし。
- **XSS対策**: メモ・タグは全てテキストとして描画（`innerHTML` に利用者入力を入れない）。
- **CSVインジェクション対策**: `= + - @` で始まるセルは出力時に `'` を付与し、取込時に除去（表計算ソフトでの数式暴発を防止しつつ可逆）。
- 保存先は端末のブラウザ内（localStorage）。**個人情報を保存しない運用**を推奨。

### 配布時の注意
- localStorage はドメイン（オリジン）ごとに分離されます。配布先URLが変わると各利用者のデータは引き継がれません（各自でCSV/JSONエクスポートしてバックアップ）。
- 改変版を再配布する場合も、CSPと「テキスト描画」の原則を崩さないでください。
